# Battle City на pygame — план разработки

Воссоздаём классику с NES (Tank 1990). Делаем итерациями: сначала
играбельное ядро (MVP), потом полировка и доп-фичи.

## Структура модулей

```
main.py                 точка входа (python battle_city/main.py)
src/
  config.py             настройки, размеры, скорости, цвета      ✅
  game.py               игровой цикл, состояния, спавн            🔨 этап 2
  menu.py               главное меню                             ✅
  sound.py              синтез звуков (двигатель/выстрел/попадание) ✅
  world/level.py        карта поля, стены, база-орёл              ✅
  entities/bullet.py    пуля                                     ✅
  entities/tank.py      танк: игрок и враг                       ✅
  entities/enemy.py     вражеский танк с простым ИИ              ✅
  entities/explosion.py анимация взрыва (большой/малый)           ✅
  entities/powerup.py   бонусы: звезда/каска/граната/лопата       ✅
```

## Этапы

### Этап 1 — Игровое ядро (MVP) ✅
- [x] Поле 13×13, отрисовка кирпича/стали/базы
- [x] Танк игрока: движение в 4 стороны (стрелки/WASD), поворот, центровка в полосе
- [x] Стрельба (пробел), кулдаун, лимит пуль игрока
- [x] Столкновение танка со стенами и границами поля

### Этап 2 — Враги ✅
- [x] Появление врагов из верхних точек, лимит на поле + всего за уровень
- [x] Простой ИИ: едет до препятствия, случайный поворот, иногда стреляет
- [x] Блокировка танк-танк (не налезают друг на друга)
- [x] Улучшена графика: танки (гусеницы/башня/ствол), кирпич, сталь, орёл, сетка

### Этап 3 — Столкновения и правила ✅
- [x] Пуля × кирпич (разрушается), × сталь (гаснет), × база (поражение)
- [x] Пуля игрока × враг (уничтожение, +очки)
- [x] Пуля врага × игрок (потеря жизни, респаун, кратковременная неуязвимость)
- [x] Встречные пули гасят друг друга
- [x] Победа: уничтожены все враги · Поражение: база разрушена или кончились жизни
- [x] Экран победы/поражения (R — заново, Esc — в меню)

### Этап 4 — Интерфейс
- [x] Главное меню: Новая игра / Загрузка / Настройки / Выход
      (активны «Новая игра» и «Выход»; остальные — заглушки)
- [x] HUD справа: очки, жизни, осталось врагов
- [x] Экран победы/поражения

### Этап 5 — Полировка
- [x] Звуки: двигатель (зацикленный), выстрел, попадание — синтез на лету
- [x] Звук взрыва/уничтожения: отдельный низкий «бум» при уничтожении
      танка/базы/гранате; лёгкий «тик» — для попаданий в стену и щит
- [x] Анимация взрыва: большой при уничтожении танка, малый при попадании
      пули в стену (покадровый процедурный спрайт)
- [x] Бонусы (power-ups): звезда (апгрейд: быстрее пуля → 2 пули → пробитие
      стали), каска (щит), граната (всех врагов с поля), лопата (рамка базы
      сталью на время). Роняют «бонусные» враги (мигают), подбор наездом

### Этап 6 — Дополнительно
- [x] Несколько уровней: карты в `levels/*.txt`, загрузчик с валидацией
      (`world/levels.py`), переход между уровнями с экраном «Уровень пройден»,
      очки/жизни переносятся; финальная победа после последней карты
- [x] Сохранение рекорда: лучший счёт в `highscore.json` (`storage.py`),
      показ в меню, HUD и на финале, плашка «Новый рекорд!» при побитии

## Технические решения
- Движение танков попиксельное (скорость 2 px/кадр при 60 FPS) с привязкой
  к сетке при поворотах — для удобной навигации по лабиринту.
- Танк (32 px) меньше клетки (40 px), чтобы проходить в проёмы.
- Разрушение кирпича — поклеточное (упрощение MVP; позже можно сделать
  «четвертинками», как в оригинале).
- Запуск: `python battle_city/main.py` (общий `.venv` в корне).
