# Bomberman

Аренный боевик на `pygame` в духе **Atomic Bomberman** (1997): сетка-лабиринт,
бомбы с крестообразным взрывом, разрушаемые блоки, бонусы и проклятия-«черепа».
Режим — **последний выживший**: один или два живых игрока против ИИ-ботов (или
дуэль друг против друга). Вся графика **процедурная** — спрайты рисуются кодом
`pygame.draw`, без внешних ассетов.

## Возможности

- **Арена 15×11** (тайл 40 px, 60 FPS): пол, несокрушимые столбы, разрушаемые
  ящики; процедурная генерация с безопасными углами.
- **Бомбы «по-взрослому»**: лимит на игрока, цепные детонации, «пинок» и бросок,
  ручной подрыв детонатором.
- **Бонусы**: Бомба+, Огонь+, Скорость, Пинок, Полный огонь, Детонатор,
  перчатка (бросок), пружина (прыжок).
- **Проклятия-черепа**: реверс управления, слишком быстрый/медленный, мини-радиус,
  авто-бомбы, «без бомб», короткий фитиль, мега-огонь. Передаются касанием
  («горячая картошка»).
- **ИИ-боты**: BFS-поиск пути, карта угроз, уклонение от пламени, атака; три
  уровня сложности.
- **Режимы**: 1 игрок против ИИ, дуэль 2 игроков, 2 игрока против ИИ.
- **Матч**: раунды «последний выживший», счёт побед (число побед до матча
  настраивается в меню, 1–9, по умолчанию 3), заставки раунда и таблица счёта,
  **sudden death** (арена схлопывается по спирали).
- **Полировка**: синтезированный звук, частицы и тряска экрана при взрыве,
  20 схем карт, спец-тайлы (телепорты, конвейеры, батуты).
- **Редактор карт** (`editor.py`) с сохранением схем в JSON.

## Запуск

```bash
source ../.venv/bin/activate
pip install -r ../requirements.txt   # если pygame ещё не установлен
python main.py
```

Требуется Python 3.12 и `pygame==2.6.1` (общий `.venv` в корне проекта).

## Управление

В меню: `↑`/`↓` — пункт, `←`/`→` — значение, `Enter` — старт, `Esc` — назад/выход.
Клавиши `1`/`2`/`3` — быстрый выбор сложности. Настраиваются: **режим**, **число
ботов** (всего ≤ 4 бойцов), **раунды** до победы в матче, **сложность** ИИ,
**карта** (случайные схемы ↔ своя), **звук** (вкл/выкл).

| Действие          | Игрок 1 (WASD)   | Игрок 2 (стрелки)  |
|-------------------|------------------|--------------------|
| Движение          | `W` `A` `S` `D`  | `↑` `←` `↓` `→`    |
| Поставить/бросить бомбу | `Пробел`   | правый `Ctrl`      |
| Ручной подрыв     | `E`              | правый `Shift`     |
| Прыжок (пружина)  | левый `Shift`    | правый `Alt`       |
| В меню            | `Esc`            | `Esc`              |

> Ввод берётся по **scancode** (физической позиции клавиши), поэтому не зависит
> от раскладки клавиатуры (RUS/ENG/…).

## Редактор карт

```bash
python editor.py
```

Кисти пол/стена/ящик/спавн/телепорт/конвейер/батут, рисование мышью
(ЛКМ/ПКМ), поворот конвейера `R`, проверка карты, сохранение/загрузка JSON,
листание карт `[` `]`. В главном меню строка **«Карта»** переключает случайные
схемы ↔ сохранённые.

## Структура

```
main.py                   точка входа и игровой цикл
editor.py                 dev-редактор схем карт
src/
  config.py               размеры сетки, тайминги, цвета
  controls.py             ввод по scancode, схемы клавиш
  sound.py                синтез звуков (16-бит PCM, без numpy)
  effects.py              частицы, тряска экрана, вспышки
  storage.py              настройки, счёт, кастомные карты (JSON)
  ai.py                   BFS, карта угроз, шаги отхода/атаки
  world/arena.py          сетка: генерация, разрушение, sudden death
  world/schemes.py        20 схем карт (узоры стен, плотность, спец-тайлы)
  entities/player.py      игрок: движение, статы, проклятия, прыжок
  entities/bot.py         ИИ-бот (наследник Player)
  entities/bomb.py        бомба: фитиль, лимит, пинок, бросок
  entities/explosion.py   крестообразное пламя, цепные детонации
  entities/powerup.py     бонусы и проклятия
maps/                     сохранённые пользовательские карты
tests/                    pytest (headless)
```

## Тесты

Чистая логика (генерация арены, геометрия взрыва, бонусы, ИИ, хранилище)
тестируется без окна (`SDL_VIDEODRIVER=dummy`):

```bash
pytest        # из каталога bomberman
```

## Технические решения

- **Кадровые 60 FPS**: движение в пикселях/кадр с привязкой к сетке.
- **Модель сетки**: клетки — естественный индекс для коллизий, взрыва и ИИ.
- **ИИ через карту угроз**: для каждой клетки — безопасна ли и сколько до взрыва.
- **Процедурная графика**: всё рисуется `pygame.draw`, без ассетов.
